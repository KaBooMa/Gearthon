Getting Started

Overview

About Gearthon

This tool provides a easy-to-use interface for both players and modders alike to mod the game. Players can install mods from modders into their mod folder and experience them in their game, just like that.

This tool utilizes GearLib to perform what it does.

Modding Possibilities

Gearthon offers a variety of tools and capabilities, including:

  • Importing OBJ files
  • Creating mesh colliders
  • Designing with custom materials
  • Utilizing ingame tweakables for fine-tuning
  • Creating versatile attachments
  • Implementing Lua scripting for advanced modifications
  • Defining and using various link types
  • And more exciting features to come!

What You Need to Get Started

To begin your journey with Gearblocks modding, you'll need the following:

  • Blender (recommended 3D modeling tool, free to use)

Getting Started

Create your first Mod

Setup (For Beginners in 3D Modeling)

  1. Download or use your preferred 3D modeling tool (we recommend Blender).
  2. Open Blender and press A to select everything in the scene, then press X to delete all selected items.
  3. Press Shift + A to add a new mesh. For this example, choose a Cube. (Set the size to 10cm)
  4. Press Tab to enter edit mode.
  5. Press A again to select every vertex (1), edge (2), or face (3).
  6. Press Ctrl + T to triangulate the mesh. (This is important because Gearblocks requires triangulated meshes.)
  7. Go to File in the top left corner, and click on Export > Wavefront (.obj).

For further guidance in Blender, check out the Blender Donut Tutorial.

Create a Mod

To create a new mod:

  1. Go to the Home page and click on Create New Mod.
  2. Enter your Mod Name and Description, then press Create.

In the Editor:

  • Click on the + next to Parts to create your first part.
  • Select the part and click on Upload Model. Choose the OBJ model you want to upload. Currently, only one mesh per part is allowed.
  • If you cannot see your model, scroll out to view it.
  • Choose a name, category, weight, mass, and other properties for your part:
    • Whether it should be paintable
    • Whether materials can be applied
    • If it should use a mesh collider

Working with attachments:

If you select Attachments under Mode or with Ctrl + 2, you can Ctrl + Left Click to create a new attachment. This is where you can attach it to other objects in the game. There are several options available:

  • Name: The name of your attachment. (Useful if you later want to reference it in lua!)
  • Attachment Flags: Flags that define how the attachment behaves:
    • linear_bearing: Allows your attachment to move linearly relative to its attachment point.
    • rotary_bearing: Allows your attachment to rotate around the attachment point.
    • constant_velocity_joint:
    • fixed: Allows the part to be rigidly connected.
    • knuckle_joint:
    • spherical_bearing: Allows your attachment to freely rotate.
  • Alignment Flags: Flags that define alignment behavior:
    • clamp_180:
    • clamp_90:
    • is_bidirectional:
    • is_female:
    • is_interior:
  • Position: The position of your attachment.
  • Orientation/Rotation: The orientation or rotation of your attachment.
  • Size: The size of your attachment.
  • Pivot: The pivot point of your attachment.

Working with Scripting:

If you select Scripting under Mode or with Ctrl + 3, you can add custom logic to your part using Lua scripting.

Create a Hopping Part

    -- Import the time module from os
    time = os.time()
    
    -- Define the delay
    delay = 1
    
    -- Define the update function, which will be triggered on every frame of the game
    function Update()
        -- Check if the player has enabled the script (E on Part)
        if not Script:IsActive() then
            -- If not active, return out of the update function
            return
        end
    
        -- Check if the delay is over
        if os.time() >= time + delay then
            -- Add 5 units of force to the y / Vertical axis of the part
            Rigidbody:AddForce(0, 5, 0)
            -- Reset the time for the next update
            time = os.time()
        end
    end
Create a Counting Cube

    -- This script will count all parts in the construction and debug the results
    -- It will only happen upon unfreezing the construction to save on performance
    -- Check your BepInEx console for the results!

    -- Define the Unfrozen function, which will be triggered every time you unfreeze your build
    function Unfrozen()
        -- Use the #Construction:GetParts() Hook to get all the connected parts
        local total_parts = #Construction:GetParts()
        -- Print out the number of total parts inside the BepInEx console
        Debug:Log("Total parts in construction is: "..total_parts)
    end

Working with Tweakables:

If you select Tweakables under Mode or with Ctrl + 4, you can add various types of tweakables:

  • Integer (non-decimal number)
  • String (text)
  • Boolean (true or false)
  • Float (decimal number)
  • Joystick action (button or range from -1 to 1)
  • Input action (keyboard input ranging from 0 to 1)

There are several options available for customization.

  • Label: The name of your Tweakable.
  • Description: The Description of your Tweakable.
  • Variable Name: The name through which you can access it in Scripting.
  • Checkbox:
Create a Tweakable Cube

For this to work you have to go to Tweakables under Mode or Ctrl + 4 and add the following 4 Tweakables:

  • whole_number - int
  • checkbox - boolean
  • decimal_number - float
  • textbox - string

    -- This script will print out what the values of our tweakables constantly.
    -- For fun, we're also multiplying our integer and float, then printing that result.
    -- Check the BepInEx console to see the results!
    
    -- You must select a value in game for this to work.
    -- Current GearLib (1.0.2 as of release) has a bug and doesn't use the default.

        Debug:Log("whole_number has a value of "..tostring(whole_number))
        Debug:Log("checkbox has a value of "..tostring(checkbox))
        Debug:Log("decimal_number has a value of "..tostring(decimal_number))
        Debug:Log("textbox has a value of "..tostring(textbox))
        
        local multiplied = whole_number * decimal_number
        Debug:Log("Numbers multiplied to the value of "..tostring(multiplied))
    end

3D Modeling

Overview

Introduction to 3D Modeling for Mods

Creating your own mods can be an incredibly rewarding experience, especially when you get to design your own 3D models. This overview will help you get started with the essentials of 3D modeling, focusing on techniques and best practices that will ensure your models are optimized for use in Gearblocks.

Understanding Mesh Structure

All 3D models are constructed from meshes, which are made up of vertices, edges, and faces. Here are some key points to remember:

  • Triangles (Tris) Only: Ensure your meshes are made up of triangles. This is crucial because triangles are the most universally compatible shape in 3D modeling and rendering.
  • Face Limit: Keep your models within a limit of 15,000 faces. This prevents artifacts.

Common 3D Modeling Tools

There are several powerful tools available for creating 3D models. Here are some of the most widely used:

  • Blender: A free and open-source 3D creation suite that supports everything from modeling and rigging to animation and rendering. It's highly recommended for beginners and professionals alike.
  • Autodesk Maya: A professional-grade software used extensively in the film, television, and gaming industries for modeling, animation, and rendering.
  • 3ds Max: Known for its modeling, animation, and rendering capabilities, this tool is popular in game development and architectural visualization.
  • SketchUp: User-friendly and great for architectural and interior design, SketchUp is perfect for quickly creating 3D models.

Best Practices for 3D Modeling

Follow these best practices to ensure your models are high-quality and optimized for use in Gearblocks:

  • Start with Simple Shapes: Begin with basic geometric shapes and gradually refine them into more complex models. This approach helps maintain clean topology and reduces errors.
  • Optimize Your Mesh: Remove unnecessary vertices and faces to keep your model efficient. This improves performance and makes your model easier to work with.
  • Use Reference Images: Always use reference images to guide your modeling process. This ensures accuracy and helps achieve realistic proportions and details.
  • Proper UV Mapping: Ensure your models have proper UV maps for texturing. Good UV mapping prevents texture stretching and distortion.
  • Regularly Save Your Work: Frequently save your progress and maintain different versions of your model. This protects against data loss and allows you to revert to previous stages if needed.

Exporting Your Model

Once your model is ready, you need to export it in a format that Gearblocks can use. Follow these steps to export your model from Blender:

  1. Select your model and ensure it is properly triangulated (press Ctrl + T in Edit Mode).
  2. Go to File > Export > Wavefront (.obj).
  3. In the export settings, ensure that "Selection Only" is checked, and set the scale to 1.0.
  4. Click Export OBJ to save your model.

Conclusion

By following this overview, you'll be well on your way to creating efficient and high-quality 3D models for your mods. With the right tools and techniques, you can bring your creative visions to life and enhance your Gearblocks experience. Happy modeling!

3D Modeling

Understanding Meshes for Gearthon

What are Meshes?

A mesh is a collection of vertices, edges, and faces that define the shape of a 3D object in Gearblocks. It serves as the fundamental component for all 3D models in the game world.

Face Limitations

Gearthon is currently limited to about 15000 faces. Models should adhere to these limitations to maintain visual quality.

Flat Normals

In Gearthon, all mesh normals will be flat. In the future we will add custom normals, so you can utilize smoothed normals.

Modeling Guidelines for Gearthon

When creating models for Gearthon, it's essential to follow certain guidelines to ensure compatibility and optimal performance:

  • Centered Models: Models should be centered at the origin point (0, 0, 0) to maintain consistency and ease of placement within the game world.
  • Optimized Geometry: Use efficient modeling techniques to minimize unnecessary geometry and keep the mesh complexity within acceptable limits.

Optimizing Meshes

To optimize meshes for Gearblocks:

  • Ensure models comply with the specified face limitations to prevent visual issues.
  • Regularly test models in Gearblocks to identify and address any issues.
  • Ensure that all visible faces have outward-facing normals! Otherwise, they will be invisible.

Conclusion

Understanding mesh limitations and adhering to modeling guidelines are essential for creating content in Gearblocks. By following these principles, you can ensure compatibility, performance, and visual quality in Gearblocks.
